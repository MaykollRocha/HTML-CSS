Sobre Portugol

O que é um pseudocódigo?

Pseudocódigo é uma forma genérica de escrever um algoritmo, utilizando uma linguagem simples sem a necessidade de conhecer qualquer sintaxe de qualquer linguagem de programação livre de contexto.

Variações

A primeira variação (dialecto) de Portugol foi criada por António Manso (Portugal) em 1986 e originou outras duas variações. O esforço iniciou como uma tradução da linguagem de programação Pascal para a língua portuguesa. Esta variação de Portugol, baseada em Pascal, atualmente é conhecida como Portugol VisuAlg, implementada em seu próprio editor por Cláudio Morgado de Souza.

História inicial

Variações inspiradas em outras linguagens de programação além de Pascal, como as linguagens C, C++ e Java. O Portugol Studio, implementado pelo núcleo de tecnologia da UNIVALI, é uma implementação baseada em Java e também escrita inteiramente em Java. Outras variações conhecidas são: G-Portugol, Portugol Viana e P&G editor.

Portugol X Pascal

O que foi o Pascal?

Pascal é a descendente direta da linguagem de programação chamada ALGOL, que era usada apenas em mainframes. Os primeiros compiladores da linguagem de programação Pascal foram criados e adaptados para microcomputadores.

Qual a semelhança?

O primeiro escritor da primeira versão do Portugol escreveu idealmente pensando que as palavras em inglês escritas no Pascal fossem apenas traduzidas para o português e usadas como Portugol.

O portugol que será abordado nessa documentação sera o portugol do Visualg

Tipagem

O Portugol é fortemente tipado, ou seja, ao definir uma variável, essa será desse tipo enquanto seu código a estiver usando e nunca mudará.

  • Tipos
  • inteiro
  • real
  • lógico
  • caracter
  • Exemplos
  • 1, 2, 3, -10, 0,...
  • 1.4, 0.5, -12.42,...
  • Verdadeiro/Falso
  • a, b, c,...

Operadores Lógicos

Server para aumentar a funcinalidade dos operadores lógicos quando você compara duas logicas ou mais para ter uma grander lógica final.

  • Lógicos
  • E
  • OU
  • NAO
  • XOU
  • Funcionalidade
  • Conjunção
  • Disjunção
  • Negação
  • XOR

Todas operações tem seu tipo de saida expecífico:

Conjunção: Resulta VERDADEIRO somente se seus dois operandos lógicos forem verdadeiros.

Disjunçã: VERDADEIRO quando um dos seus operandos lógicos for verdadeiro.

Negação: nao VERDADEIRO = FALSO, e nao FALSO = VERDADEIRO

XOR: VERDADEIRO se seus dois operandos lógicos forem diferentes, e FALSO se forem iguais.

Laços de Repetição

Os lalos de repetição atualmente são 3, for,while e do..while porem muitas linguagem não tem o do..while, essa pseudolinguagem tem e por isso vou mostrar elas. Os loços são importante para deixar seu codigo em loop até quando quiser e criar ali oque quiser, aindan usando um switch e futuramente trabalhar com funções e procedimentos.

repita..até

As condições varia de ponto a ponto, e como utilizar cada laço vai do caso, um caso simples para faça..até e quando você tem que entregar algo que presciso ver primeiro a entrada que a comparação como receber uma senha ou algo assim.

algoritmo "repita..até"

var

senha : caracter

inicio

repita

ler(senha)

enquanto senha != "1234214"

fimalgoritmo

enquanto..faça

O laço enquanto prescisa respeitar as condições inicias para entrar no laço, assim como o if que você percisa de uma condição para entrar, mas o while ou enquanto não possui um condição reversa. Para isso você coloca sua variavel de entrada com uma condição que respeite o inicio dela.

algoritmo "enquanto..até"

var

i : inteiro

inicio

i <- 1

enquanto i <= 10

escreva(i)

i <- i + 1

fimenquanto

fimalgoritmo

para..faça

Para é muito usado para diversas atividades como intergir com laçõs lineares, vatores ou fazer algo que prescise de forçã bruta enquanto temos isso usaremos, no futuro quando aprender funções teremos recursão.

inicio
